# WoW 信息实时查询

当回答魔兽世界游戏机制相关问题时，不要依赖自身训练数据中的旧信息。通过外部数据源获取当前版本的准确数据。

## 核心原则

**永远假设你的内置知识可能过时。** 12.0 至暗之夜是全新资料片，所有职业都经历了重制，天赋树全部重做，Boss 机制全是新的。不要用旧版本的知识回答新版本的问题。

## 触发条件

以下场景必须触发外部查询：

- Boss 机制 / 副本攻略（"XX怎么打""XX Boss 机制是什么"）
- 天赋推荐 / 构建方案（"XX职业怎么点天赋""现在什么专精强"）
- 装备推荐 / BiS（"XX位置带什么""毕业装是什么"）
- 附魔 / 宝石 / 消耗品（"附魔选什么""吃什么buff食物"）
- 补丁/热修变动（"最近改了什么""XX技能被砍了吗"）
- 大秘境词缀（"这周什么词缀""词缀怎么处理"）
- 数值类问题（具体伤害数字、CD 时间、百分比等）

## 数据源与查询策略

按以下优先级查询。**不要一次查所有源**，先查最可能命中的，没找到再查下一个。

### 1. WoWHead（首选，最全面）

用途：物品、技能、天赋、Boss 能力、任务、成就
```
查询方式：搜索 wowhead.com 相关页面
示例查询："site:wowhead.com 虚影尖塔 攻略" 或 "wowhead midnight raid guide"
```

中文信息不全时，查英文版再翻译要点。

### 2. NGA 玩家社区（中文攻略首选）

用途：中文攻略、职业讨论、国服特有信息
```
查询方式：搜索 NGA 相关帖子
示例查询："site:bbs.nga.cn 至暗之夜 虚影尖塔 攻略"
```

NGA 的帖子质量参差不齐，优先看精华帖和高回复帖。

### 3. Icy Veins（职业指南）

用途：天赋推荐、属性优先级、循环手法
```
查询方式：搜索 icy-veins.com 对应职业页面
示例查询："icy-veins fury warrior midnight guide"
```

Icy Veins 的指南通常在版本上线后 1-2 周内更新完毕。

### 4. Raider.io（大秘境数据）

用途：大秘境分数、热门构建、地下城排行
```
API 端点（如果可用）：
GET https://raider.io/api/v1/characters/profile?region=cn&realm=血色十字军&name={角色名}&fields=mythic_plus_scores_by_season:current
```

### 5. Warcraft Logs（团本数据）

用途：DPS/HPS 排行、Boss 击杀数据、Log 分析
```
查询方式：搜索 warcraftlogs.com 对应数据
```

### 6. 暴雪官方（蓝贴/热修）

用途：版本公告、热修内容、已知 Bug
```
查询方式：搜索 wow.blizzard.cn 或 worldofwarcraft.blizzard.com 官方公告
```

## Instructions

### 回答攻略类问题

1. 先判断问的是哪个副本/Boss/职业
2. 搜索 WoWHead 或 NGA 获取当前版本攻略
3. 提取关键机制要点，用简洁的语言重新组织
4. 标注信息来源和适用版本（如"根据 12.0 第1赛季数据"）
5. 如果信息可能在近期热修中有变动，加一句提醒

### 回答天赋/构建问题

1. 搜索 Icy Veins 或 WoWHead 的职业指南
2. 区分场景推荐（大秘境 vs 团本 vs PvP）
3. 说明推荐理由，不只是列天赋点
4. 提醒"天赋每个补丁可能调整，以游戏内最新为准"

### 回答装备/附魔问题

1. 搜索当前版本的 BiS 列表
2. 说明属性优先级和原因
3. 对于数值敏感的推荐（如"暴击到多少后转急速"），建议用 Raidbots 模拟

### 找不到最新信息时

老实说"这个信息我暂时查不到最新版本的数据"，然后：
- 给出最可能找到答案的网站链接
- 如果有旧版本的参考信息，可以提供但明确标注"这是旧版本数据，仅供参考"

## Rules

- 查到的信息必须标注来源和时效性
- 不把多个来源的信息混在一起说成是同一个来源的
- 英文来源的内容翻译成中文时，保留关键技能名的英文（便于游戏内对照）
- 每次查询控制在 2-3 个搜索以内，别为了一个问题查 10 遍
- 如果群里有人在讨论当前热修内容且你不确定，建议去看蓝贴而不是猜
- 搜索结果如果是 11.x（地心之战）版本的，要明确告知用户这是上个版本的信息

## 版本缓存机制

为减少重复查询，可以将以下信息缓存到 memory.md（标注缓存日期）：

- 当前赛季的词缀轮换表（每赛季更新一次）
- 各职业当前版本的主流天赋方向（每个补丁检查一次）
- 当前团本 Boss 列表和基本机制要点（首次查询后缓存）

缓存条目格式：
```
[WoW缓存 YYYY-MM-DD] 内容描述
```

超过 14 天的缓存视为可能过期，再次被问到时重新查询。
