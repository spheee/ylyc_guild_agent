# USER.md — 运营者档案

这份文件描述「雨落烟尘-robot」的运营管理者，供 Agent 在互动中做出更准确的判断。

---

## 基本信息

| 字段 | 内容 |
|------|------|
| 角色 | 公会技术管理员 / Bot 运营者 |
| 游戏身份 | 「雨落烟尘」公会成员（具体角色名见 guild_config.json contacts） |
| 主要职责 | 维护 Bot 配置、处理技术问题、更新公会数据 |
| 联系方式 | 见 guild_config.json bot_config.admin_qq_ids |

---

## 工作模式

**何时出现：**
- 更新公会配置（团长变更、进度更新、活动时间调整）
- 排查 Bot 异常行为
- 添加/修改 Bot 功能
- 重大公告前的准备

**指令风格：**
- 倾向于直接给出明确指令，不需要多余解释
- 偏好简洁回复，不喜欢废话

---

## 管理权限

运营者拥有完整的 Bot 管理权限，包括：

- 修改 `guild_config.json` 中所有字段
- 读写 `guild.db` 数据库
- 修改 `SOUL.md` / `IDENTITY.md` / `AGENTS.md`
- 开关定时任务（HEARTBEAT.md 中的任务）
- 查看 `bot_logs` 中的异常记录

---

## 对 Bot 的期望

- 出现配置问题时，直接告知具体是哪个字段、哪个文件出了问题
- 技术性问题用技术性语言说，不需要像对普通群员那样简化
- 如果 Bot 检测到需要关注的异常，主动告知而不是等被问到
- 数据库操作前确认，执行后汇报结果

---
