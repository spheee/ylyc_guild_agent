# HEARTBEAT.md — 定时任务配置

定义「雨落烟尘-robot」的定时触发任务。各任务在 OpenClaw 或宿主平台的调度器中配置，按 cron 表达式执行。

---

## 任务列表

### 1. 活动日开团提醒

**触发时间：** 每周一、周二、周四 **19:30**（活动前30分钟）

**Cron：** `30 19 * * 1,2,4`

**执行内容：**

```
读取 guild_config.json 确认今天是活动日
→ 发送到主QQ群：

"[提醒] 今晚 8 点开团，还有 30 分钟！
未报名的同学赶紧群里说一声，或者回复'报名'。
当前已报名人数：[查 guild.db 当天数据]"
```

**静默条件：** 如果当日群内已有管理员发布活动取消通知，跳过本次提醒。

---

### 2. 活动日报名截止提醒

**触发时间：** 每周一、周二、周四 **19:55**（活动开始前5分钟）

**Cron：** `55 19 * * 1,2,4`

**执行内容：**

```
查询今日报名情况
→ 如报名人数 < 10，发送：

"[最后召集] 离开团还有 5 分钟！
目前报名：[已报名人数]人（[T/奶/DPS各几个]）
还有人要来的赶紧说！"

→ 如报名人数 >= 10，发送：
"[即将开团] 今晚人数够了，8 点准时见！已报名 [N] 人。"
```

---

### 3. 次日活动预告

**触发时间：** 每周日、周一、周二 **22:00**（前一晚提前预告）

**Cron：** `0 22 * * 0,1,2`

**执行内容：**

```
判断明天是否是活动日（周一/周二/周四）
→ 是，发送：

"[预告] 明天（周X）晚上 8 点有团，感兴趣的提前报名吧。"
```

**注意：** 仅在活动前一天触发，不重复提醒。

---

### 4. 每周进度播报

**触发时间：** 每周五 **20:00**（本周团本结束后）

**Cron：** `0 20 * * 5`

**执行内容：**

```
读取 guild_events 中本周的 first_kill 类事件
读取 guild_config.json 中各团的 current_progress
→ 发送本周小结：

"[本周战报]
一团：[本周首杀情况或进度更新]
二团：[本周首杀情况或进度更新]
下周继续冲！"
```

**静默条件：** 如果本周无进度更新，跳过（不发无意义的"本周无首杀"）。

---

### 5. 词缀提醒（可选）

**触发时间：** 每周二 **09:00**（大秘境周重置日）

**Cron：** `0 9 * * 2`

**执行内容：**

```
触发 wow-lookup 技能查询本周大秘境词缀
→ 发送到主QQ群：

"[周常重置] 本周大秘境词缀：[词缀1] / [词缀2] / [词缀3]
[简短的词缀应对建议，1-2句]"
```

---

## 配置说明

| 参数 | 说明 |
|------|------|
| `timezone` | 所有时间均为 `Asia/Shanghai`（UTC+8） |
| `target_group` | 消息发送到 `qq_group.main_group_id` |
| `skip_if_admin_announced` | 检测到管理员当日发布了相关公告时，跳过重复提醒 |

## 启用/禁用任务

在宿主平台（OpenClaw / 任务调度器）中按任务名称开关：

- `raid-reminder-30min` — 任务1
- `raid-reminder-5min` — 任务2
- `raid-preview` — 任务3
- `weekly-progress` — 任务4
- `weekly-affix` — 任务5
